# Global UI Interaction And Project Context - Pre-Fix Analysis

Ngày audit: 2026-06-30

## Phạm vi đã kiểm tra

- Global search: `src/components/layout/global-search-command.tsx`, `src/components/layout/header.tsx`, `src/components/layout/app-shell.tsx`, Dashboard page/component, action `src/app/actions/global-search.ts`.
- Project selector/current project: `src/components/layout/global-project-context-switcher.tsx`, `src/lib/project-context.ts`, Dashboard, Reports, Documents, Materials, Contracts, Accounting/Payments, Approvals.
- Project status mapping: Prisma enum `ProjectStatus`, seed, Project list/detail, Dashboard overview, Topbar project selector.
- Drawer/Dialog/Popover: report drawer/gallery/create dialog, document viewer/upload/rename/edit dialogs, material dialogs, contract/payment/approval detail drawers, notification/help/project popovers, shared confirm/reason dialogs.
- Next.js internal docs read before code changes: layouts preserve state across navigation, server/client boundaries, search params, preserving UI state with Activity.

## Root Cause Summary

Next.js layout/client state can be preserved across route changes. Current transient overlays mostly rely on local `useState` and do not consistently close/reset on `pathname`/`searchParams` changes. Several modules also derive project context from the URL/cookie only on initial render, or fallback to the first project when no selected project exists. Project status display is duplicated in multiple components, so `PLANNING` can be omitted from data queries or shown under active/working UI language.

## Hypothesis Trước Khi Fix

1. Lỗi search Dashboard không phải do build/tsc mà do state overlay trong shared layout không reset theo route/lifecycle.
2. Lỗi project context không đồng nhất do từng module tự quyết định project scope: có module lấy URL/cookie, có module chỉ client-filter, có module fallback project đầu tiên.
3. Lỗi "công tác chuẩn bị" bị nhìn như "đang thi công" do query chỉ lấy `ACTIVE` ở các vùng overview và mapping status không tập trung.
4. Drawer/dialog bị kẹt là lỗi lifecycle: thiếu route-change cleanup, thiếu ESC/body-scroll cleanup ở một số drawer.

## Acceptance Gates Sau Fix

- Static QA guard fail nếu text search cũ xuất hiện trong production source.
- Static QA guard fail nếu Materials còn fallback `projects[0]` khi không có `projectId`.
- Static QA guard fail nếu core drawers thiếu ESC hoặc outside close.
- `npx tsc --noEmit` pass.
- `npm run build` pass.
- `npm run lint` chạy nếu dự án hỗ trợ ổn định; nếu fail phải ghi nguyên nhân thật.
